# PynqCコンパイラABI

CPUの[初期ABI](abi.md)を変更せず利用する。`r1`は戻り値、`r2`〜`r5`は最大4引数、
`r6`〜`r10`はcaller-saved、`r11`〜`r13`はcallee-saved、`r14`はLR、`r15`はSPである。

## フレームポインタ

`r13`をフレームポインタ（FP）として予約する。式評価によってSPが一時的に上下してもローカル
オフセットを固定でき、可読性と再帰の安全性が高いためである。callee-saved規則に従って保存する。

## フレーム配置

```text
高位アドレス
    呼出し元フレーム
    保存LR             FP + 4
    保存FP             FP + 0
    引数退避・ローカル FP - 1以下
    ローカル配列
    式評価spill        現在のSPから下へ動的にpush
低位アドレス
```

プロローグはSPを8 byte下げ、旧FPとLRを保存し、FPへSPを複写してから4 byte境界へ丸めた固定
フレームを確保する。引数レジスタは対応するローカルスロットへ直ちに保存する。エピローグは
SPをFPへ戻し、FP/LRを復元して`RET`する。全関数で同じ形を使うためleaf判定は不要である。

`int`とポインタは4 byte整列、`char`は1 byte整列、フレーム全体は4 byte整列する。配列は
連続配置する。現在は全ローカルの生存期間を関数全体とみなし、スコープ終了後のスロット再利用は
行わない。

## 呼出しとspill

引数を左から評価して各値をスタックへpushし、逆順にpopして`r2`〜`r5`へ配置する。
戻り値`r1`は`r6`へ移す。二項演算の左辺やネストした呼出しを跨ぐ値はスタックへspillするため、
caller-savedレジスタの暗黙上書きはない。callee-savedの`r11/r12`は通常コード生成で使わず、
ランタイムが使う場合だけ明示保存する。

再帰では各呼出しが独立したFP、保存LR、ローカル領域を持つ。スタックあふれ検査は未実装であり、
グローバル領域と下向きスタックの衝突防止は今後の課題である。
