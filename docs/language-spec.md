# PynqC言語仕様

PynqCは独自32 bit CPUの教育・OS開発用に限定したC-like言語である。C17との互換性は
目標にせず、未対応構文を暗黙に受理しない。

## 字句

識別子はASCII英字または`_`で始まり、後続にASCII英数字と`_`を置ける。空白、改行、
`//`行コメント、`/* ... */`ブロックコメントを無視する。ブロックコメントはネストしない。
キーワードは`int char void if else while for return break continue sizeof`である。

10進整数と`0x`/`0X`付き16進整数を32 bitで読む。負数は負整数リテラルではなく単項`-`と
整数として構文解析する。文字と文字列は`\n \r \t \0 \\ \' \"`を扱う。文字列は
データRAMへ末尾NULを付けて配置する。

## 型

| 型 | サイズ | 意味 |
|---|---:|---|
| `int` | 4 | 2の補数32 bit符号付き整数 |
| `char` | 1 | 0〜255の符号なし8 bit整数 |
| `void` | なし | 値を持たない |
| `T*` | 4 | データRAMのbyteアドレス |
| `T[N]` | `sizeof(T)*N` | 固定長1次元配列 |

条件では0が偽、それ以外が真である。`char`は式で`int`へ変換できる。配列は`sizeof`と
配列初期化を除く値文脈で先頭要素へのポインタへ変換する。ポインタ加減算は指す要素の
サイズでスケールする。同じ要素型のポインタ同士だけを比較・減算できる。

## 宣言とスコープ

ローカル変数、グローバル変数、ブロックスコープ、シャドーイングを扱う。同一スコープの
重複定義はエラーである。宣言は1文につき1変数とし、複雑なC宣言子は扱わない。
グローバル初期値は整数、文字、文字列アドレス、char配列の文字列初期化に限定する。
初期値なしグローバルは`_start`がゼロ初期化する。ローカル変数の未初期化読出し検査は
現段階では行わない。

## 演算子

強い順に、後置`[] () ++ --`、単項`+ - ! ~ & * ++ -- sizeof`、`* / %`、`+ -`、
`<< >>`、`< <= > >=`、`== !=`、`&`、`^`、`|`、`&&`、`||`、代入を適用する。
代入は右結合、それ以外の二項演算は左結合である。`&&`と`||`は短絡評価し、比較・論理の
結果は0または1である。`>>`は符号付き算術右シフトである。

代入は`= += -= *= /= %= &= |= ^= <<= >>=`、前置・後置の`++ --`を扱う。整数演算は
32 bitでラップする。乗算は`__mulsi3`、除算と剰余は`__divsi3`と`__modsi3`へ下げる。
ゼロ除算はSIM_EXITへ1を書いて停止する。

## 文と関数

空文、式文、ブロック、`if/else`、`while`、`for`、`return`、`break`、`continue`を扱う。
`break`と`continue`はループ内に限る。関数は宣言、定義、前方宣言、0〜4引数、再帰、
`void`戻りに対応する。非void関数は保守的な明白return検査を通る必要がある。

プログラム入口は引数なしの`int main()`である。ランタイム関数は宣言を書かなくても
組み込み宣言として型検査される。

## 配列、ポインタ、sizeof

ローカル／グローバル配列、添字、`&`、`*`、ポインタ代入と加減算を扱う。`char text[] =
"..."`ではNULを含む長さを推論する。多次元・可変長配列は未対応である。
`sizeof(int)`、`sizeof(char)`、`sizeof(pointer)`、`sizeof(expression)`を扱い、式は実行しない。

## Cとの差異と未対応機能

プリプロセッサ、複数翻訳単位、明示キャスト、`struct/union/enum/typedef`、修飾子、記憶域指定、
関数ポインタ、5個以上の引数、可変長引数、浮動小数点、`switch/goto/do`、複合初期化子、
カンマ演算子、条件演算子は未対応である。`char`は常にunsignedであり、右シフトは`int`の
算術右シフトに固定する。
