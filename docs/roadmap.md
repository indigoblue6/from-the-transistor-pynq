# 開発ロードマップ

## コンパイラ基盤

1. データRAM初期化イメージと独自オブジェクト形式を定義する。
2. シンボル、セクション、再配置を扱うリンカと複数翻訳単位を追加する。
3. 基本ブロックと仮想レジスタを持つIR、続いてSSA IRを導入する。
4. 定数畳込み、不要コード削除、レジスタ割当てを段階的に追加する。
5. ソース行・シンボルを使うエミュレータ／ILAデバッガを作る。

## CPUとOS

1. 割り込み入口、原因・復帰PCレジスタ、割り込み禁止制御をISAへ設計する。
2. メモリマップドタイマーとGPIOを実装し、PYNQ-Z1実機で検証する。
3. カーネル／ユーザーモード、例外、システムコールABIを定義する。
4. コンテキスト保存、プリエンプティブスケジューラ、マルチタスクを実装する。
5. RAMアロケータ、UARTドライバ、簡易ファイルシステムを追加する。
6. OSカーネルの主要部分をPynqCへ移植し、ブートストラップを成立させる。

CPU/FPGA変更はRTLシミュレーションだけで完了とせず、bitstream生成、PYNQ-Z1書込み、ILAまたは
物理USB-UART観測まで行う。外付けUSB-UARTケーブルがない間はJTAG/ILAを必須経路とする。
